# Kaspersky user-app
## Установка
>После клонирование репозитория
```bash
cd server
npm install

cd ../frontend
npm install
```
## Запуск
Сначала запускаем сервер
```bash
cd server
npm run dev # http://localhost:3000
```
Затем фронт
```bash
cd frontend
npm start # http://localhost:3001
```

## Выводы по самостоятельному проектированию и с использованием LLM
### Самостоятельное проектирование
Главный плюс - это полный контроль над процессом.
При самостоятельном проектировании разработчику в первую очередь проще менять и переиспользовать созданную логику, а также реализовывать виденье UI
### Использование LLM на примере групп пользователей
При использовании LLM в первую очередь достаточно трудно объяснить, как именно ты видишь свое приложение.
Модели приходится ставить четкие границы, где она может работать и по каким примерам создавать, а где нельзя модифицировать логику и нужно спрашивать всякий раз, когда ей хочется что то поменять
Без понимания и использования исключительно LLM в течении всего процесса разработки произойдет хаос, в котором будет тяжело ориентироваться и дорабатывать
Но если грамотно использовать данную возможность, то модель превращается из "копирки и основы", в полезный инструмент, который при должном умении сократит время разработки без потери качества

## Основные возможности
* Просмотр пользователей - таблица со всеми пользователями и их данными
* Сортировка - по имени и группе (клик на заголовки столбцов)
* Поиск - по имени, email и телефону
* Добавление пользователей - форма с валидацией полей
* Удаление пользователей - множественное удаление через чекбоксы
* Просмотр групп - статистика по группам с количеством участников

## API Endpoints
```
GET /api/users - получение всех пользователей
POST /api/users - добавление нового пользователя
POST /api/users/delete - удаление пользователей (массив ID)
```
